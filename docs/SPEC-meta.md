# SPEC — 养成/元系统升级（两款游戏共用，各自实现）

给 7 岁的 Leon 做的养成循环，对标 Paper.io 2 的「金币→宝箱→收集皮肤→段位晋升→赛后庆祝」，
但**全部免费、无广告、无内购、无付费点**。核心设计铁律（来自儿童激励研究，必须逐条遵守）：

## 0. 儿童安全铁律（硬性，写进代码约束）
- **只增不减**：金币、收藏数、段位、纪录永远只涨，死亡/输局绝不扣任何已得的东西。
- **无损失厌恶**：一局只会「加」，绝不「减」。
- **无 FOMO / 无限时**：没有「限时 24h」，所有皮肤永远可得，无轮换商店、无赛季消失物。
- **无锁**：没有体力/生命/冷却锁，随时能立刻再来一局。
- **无签到愧疚**：每日礼物只作为「惊喜礼物」，缺席零惩罚、绝不显示「连击中断」。
- **慷慨**：掉率与获取速度调到「几乎每局都解锁点东西，几周休闲玩能集齐全部」。
- **保底**：宝箱保底——每开到第 5 个宝箱必出一个未拥有的新皮肤（若还有未拥有的）；软保底：距上次出新皮肤越久，出新皮肤概率越高（40%起，每箱+15%，第5箱100%）。绝不让孩子连开一堆全是金币的箱。
- **皮肤纯装饰，零属性加成**（杜绝 pay-to-win / collect-to-win 压力）。
- **图标优先、文字极简**：每个操作看图标就懂；文字只作短补充。
- **隐形照顾**：连输 2–3 局后，悄悄降低 bot 难度/数量（绝不显示「难度已降低」）。

## 1. 模块结构（每款游戏各自内含一份，保证 PWA 离线正确）
> 关键：Service Worker 作用域限于各自目录，`../shared/` 在作用域外无法离线缓存。
> 因此 meta 代码**复制进每个游戏目录**，用「按游戏配置」区分，不跨目录共享。

每款游戏新增：
```
js/meta/
  meta.js        # 存档 schema + 纯函数经济引擎（金币/宝箱/保底/段位）+ Meta 门面
  meta-config.js # 本游戏的配置：皮肤目录、金币公式、段位表、生日配置
  meta-ui.js     # 覆盖层 UI：主菜单钱包/段位牌、收藏册、宝箱开箱、赛后结算流程
  meta-fx.js     # 彩带、计数动画、光爆等一次性特效（用 CSS/canvas，非游戏热循环）
css/meta.css     # 养成系统所有覆盖层样式（大圆角、大按钮、rarity 配色、安全区）
```
`main.js` 通过 `Meta` 门面交互，绝不直接碰 localStorage。

## 2. 存档 schema（localStorage，单键 JSON，版本化+抗损坏）
键：`<prefix>.meta`（paper=`quanland.meta`，snake=`noodle.meta`）。旧的分散键（best/skin/kills/games/muted）**迁移**进来后保留兼容读取。
```js
{
  v: 1,                         // schema 版本，用于迁移
  coins: 0,                     // 金币（只增显示 lifetimeCoins 另存）
  lifetimeCoins: 0,             // 累计赚取，永不减（展示用）
  trophies: 0,                  // 段位星，只增
  skins: { icecream:{owned:true, isNew:false}, ... }, // 每皮肤拥有/NEW 标记
  selectedSkin: 'icecream',
  pity: { sinceNew: 0 },        // 距上次出新皮肤的开箱数（保底）
  daily: { lastClaimed: 0 },    // 每日礼物时间戳
  best: 0, kills: 0, games: 0,  // 迁移自旧键
  birthdaySeenYear: 0,          // 生日彩蛋当年是否已展示（避免重复轰炸）
}
```
- 读取时 **schema 校验 + 兜底**：任何字段缺失/类型错/负值 → 用默认值/clamp，绝不抛错、绝不崩。
- 全部 try/catch（隐私模式/配额）。写入节流（每次结算/开箱后 flush 一次）。
- 迁移：首次加载若无 `.meta` 但有旧键 → 组装新 schema（best/kills/games/muted/skin 继承），并按 best 把旧「按分数解锁」的皮肤标记为 owned。

## 3. 经济引擎（纯函数，注入 RNG，可单测）
用 mulberry32 种子 RNG（生产用时间种子，测试用固定种子）。所有算钱/掉落/保底为纯函数。

### 3.1 每局金币
- paper：`coins = 10 + round(pct*8) + kills*25 + (won?100:0)`，上限 500。
- snake：`coins = 10 + floor(length/6) + kills*20 + (top3?100:0)`，上限 500。
- 大而整的数字才「爽」；用 `128.34K` 式紧凑格式（Intl.NumberFormat compact，<1000 显示精确整数）。

### 3.2 段位星（只增，绝不减）
按本局名次给星：第1名 +30，第2 +20，第3 +15，前半程 +8，其余 +5。累计 → 段位。
段位表（含 emoji，图标优先）：
`🌱 新手(0) → 🥉 青铜(60) → 🥈 白银(180) → 🥇 黄金(400) → 💎 钻石(800) → 👑 大师(1500) → 🌈 传奇(3000)`
跨过边界 → 触发「段位晋升」庆典覆盖层（§6）。

### 3.3 宝箱
- **来源（全部免费）**：每局结束必得 1 个；每日礼物 1 个（§3.5）；金币可在收藏册直接买皮肤（金币出口）。
- **档位**（按本局表现加权，越好越可能高档）：`木箱 / 银箱 / 金箱 / 彩虹箱`。
  表现分 = 金币公式的同一个 perf；perf 高则权重偏向高档。木60/银28/金10/彩2 起调，perf 高时右移。
- **内容**：金币（按档位区间）+ 皮肤 roll。皮肤 roll 用保底：
  - 若 `sinceNew+1 >= 5` 且仍有未拥有皮肤 → 强制出一个未拥有的新皮肤；否则按软保底概率（`min(1, 0.40 + 0.15*sinceNew)`）尝试出新皮肤；命中则从未拥有池按稀有度加权抽一个。
  - 出新皮肤：`owned=true, isNew=true`，`sinceNew=0`；否则 `sinceNew++`，把该 roll 折算成额外金币（「多得金币！」，绝不出现「你已拥有」的沮丧）。
  - 全部皮肤集齐后保底自动失效，箱子只给金币，永不卡死。

### 3.4 皮肤目录 + 稀有度 + 金币直购
稀有度四档配色：常见灰 `#9e9e9e` / 稀有蓝 `#4a90d9` / 史诗紫 `#9b59b6` / 传说金 `#ffb300`。
金币直购价：常见 200 / 稀有 600 / 史诗 1500 / 传说 4000（几局就能买一个；宝箱保底也会免费送）。
收藏册里未拥有皮肤显示为「神秘剪影 + ?」（不是挂锁，别读成「付费」），可点「用金币解锁」。

### 3.5 每日礼物（纯惊喜，零愧疚）
新的一天首次进主菜单 → 吉祥物递上一个免费箱（框架为「送你的礼物！」）。**不显示可中断的连击计数**，缺席零提及。冷却 20h（「第二天早上」就绪）。时钟被调回（now<last）→ 视为未就绪，不发放、不崩。

### paper（emoji 角色）

## 5. UI（覆盖层，HTML/CSS，非游戏热循环；一次性 CSS 动画可用 box-shadow/filter）
所有覆盖层大圆角、大按钮（≥56px）、`touch-action:manipulation`、`env(safe-area-inset-*)` 内边距、`user-select:none`。数字用计数上滚动画（easeOutCubic）。

### 5.1 主菜单增强
顶部一行钱包：`🪙 金币(紧凑格式)` + 段位徽章（emoji+名+进度小条「距下一段 xx★」）。
按钮区新增：`🎁 收藏册`（开收藏 gallery）、每日礼物图标（就绪时红点+跳动）。皮肤选择改为从收藏册选（或保留菜单里横排已拥有皮肤快选）。

### 5.2 收藏册 gallery（CSS grid）
- 顶部大字「已收集 X / N」+ 进度条。
- 每格：拥有=彩色皮肤+稀有度描边+NEW 角标（首次拥有，进册查看后清除）；未拥有=剪影+?+「🪙 xxx 解锁」按钮（金币够则可买）。
- 稀有度用**颜色+边框+标签**三重区分（色盲友好）。选中已拥有皮肤 = 设为当前皮肤。

### 5.3 宝箱开箱序列（§研究 CSS）
出现 → 抖动 2–3 次（tick 音渐强）→ 白光爆 + 彩带 → 内容飞出放大居中 →「获得！」。皮肤为新 → NEW 角标 + 更多金色彩带 + 欢呼音。传说档背后加旋转 conic 光芒。整个 1–2 秒，短而有仪式感。可连续开多箱（每日+赛后）。

### 5.4 赛后结算流程（async 顺序编排，promise/await）
死亡/结束后依次（每步 await，可点击加速跳过）：
1. 面板滑入，展示本局成绩（%/长度、击杀、名次、最高纪录；破纪录→「新纪录!」+彩带）。
2. 金币计数上滚（钱包从旧值滚到新值）。
3. 段位星条填充；若跨段 → 段位晋升庆典（§6）。
4. 若得箱 → 宝箱开箱序列（§5.3），内容入账。
5. 大「再来一局」（1 键 <1s 重开）+ 小「收藏册」「回主菜单」。
- 用 try/finally 保证任何动画异常都不会把孩子卡在黑覆盖层里。

### 5.5 段位晋升庆典（§6）
暗背景淡入 → 旧徽章缩灰 → 新徽章弹簧放大+白光爆+彩带 → 段位名滑入（该段配色光晕）→ 点击继续。

## 6. 特效（meta-fx.js）
- `confettiBurst({count,origin,gold})`：全屏临时 canvas 粒子（速度+重力+阻力+旋转），自动清理；传说/晋升时金色更多。
- `countUp(el, from, to, {fmt})`：rAF + easeOutCubic + 紧凑格式。
- 光爆、旋转光芒用 CSS keyframes。

## 7. 生日模式（Leon）
配置在 `meta-config.js`：`BIRTHDAY = { name:'Leon', month:?, day:?, windowDays:1 }`。
> ⚠️ 具体月/日由 Nick 提供后填入；先用占位并清晰标注 TODO。windowDays=1 表示只当天（可设 ±N 天窗口容错）。
- **触发**：进主菜单时，若今天在生日窗口内且 `birthdaySeenYear != 今年` → 播放生日彩蛋，然后置 `birthdaySeenYear=今年`（当天只轰炸一次，之后进菜单只显示轻量生日横幅）。
- **彩蛋内容**：
  1. 满屏金色彩带 + 生日音乐小段（WebAudio 合成「生日快乐」旋律）。
  2. 大字横幅「🎂 Leon，7 岁生日快乐！」（年龄可由出生年推算或直接写）。
  3. 赠送生日限定皮肤 🎂（owned=true, isNew=true）+「送你的生日礼物！」开箱式呈现。
  4. 主菜单当天带生日主题点缀（气球/彩带装饰、生日横幅常驻当天）。
- **玩家名**：排行榜/结算/名牌里玩家显示为 **Leon**（替换原来的「你」/「You」），非生日日也一直显示（个人化）。生日当天名字旁加 🎂。
- 生日彩蛋纯正向、无任何催促。

## 8. 游戏内增强（两款都要）
- **连杀**：短时间窗口（~3s）内连续击杀 → 升级弹窗「双杀!」→「三杀!」→「超神!」（4+），大字+音效+额外金币（双杀+30/三杀+80/超神+150，计入本局金币），窗口内每杀刷新计时。
- **玩家名 Leon**（见 §7）。
- **隐形照顾难度**：`meta` 记录最近连败；连败 ≥2 → 下一局 bot 数-1 或激进度-15%（隐形，绝不提示）；连胜则缓回默认。paper 接 botCount/aggression，snake 接 botCount/HUNT 概率。
- 名次/胜负定义（喂给结算）：paper 用最终 % 在本局所有玩家中的名次 + 是否 100%；snake 用最终最高名次 + 是否 top3。

## 9. 与现有代码的接线点（已确认）
- paper：`main.js` 里 `gameOver()`/`doVictory()` 结算处调 `Meta.reportMatch({pct,kills,rank,won})` 并跑结算流程；`buildSkins()` 改为读 Meta 的已拥有皮肤；`game.js` 击杀回调接连杀；bot 生成读 Meta 难度。玩家 `name='Leon'`。
- snake：`main.js` 里 `onDeath()` 调 `Meta.reportMatch({length,kills,rank,top3})`；`buildSkins()`/`SKINS` 改为 Meta 目录；击杀 toast 处接连杀；bot 生成读难度。玩家名 Leon。
- 核心玩法逻辑（移动/碰撞/占领/渲染）**不改**，元系统纯叠加。
